# タイムボム MVP

対面プレイ用『タイムボム』の Web アプリ MVP です。  
プレイヤーは各自のスマホを使い、秘密情報は自分の端末だけで確認します。

仕様の基準は [docs/timebomb_webapp_mvp_spec.md](C:/Users/taise/dev/タイムボム/docs/timebomb_webapp_mvp_spec.md) です。

## 実装方針

- 真実のゲーム状態はサーバーのみが保持
- クライアントには公開情報と本人の秘密情報だけを送信
- リアルタイム同期は WebSocket ベース
- 永続化なしのインメモリルーム管理
- 同端末リロード復帰は `roomCode + playerId + sessionToken` で実施

## 現在の実装範囲

- ルーム作成
- 参加コードによる参加
- 名前重複拒否
- ロビー
- ゲーム開始
- 役職配布
- 導線配布
- 役職確認 / 導線確認
- ホストによる配り直し
- メインゲーム進行
- サーバー側勝敗判定
- 同端末リロード復帰
- 終了画面での全役職公開

- WebSocket サーバー: `ws://localhost:3001`

### ビルド確認

```bash
npm run build
```

### テスト

```bash
npm test
```

## 画面と挙動

- ルート画面で名前入力、ルーム作成、参加コード参加が可能
- ルーム参加後は URL に `roomCode` を保持し、同じブラウザで再読込すると自動復帰を試行
- 役職確認と導線確認は全員の確認完了で次フェーズへ進行
- メイン画面では未公開カードは裏面表示のみ
- 導線内容の再確認は本人端末のモーダルでのみ可能
- カード切断は対象プレイヤー本人の端末からのみ送信可能

## テストで確認しているケース

- 4人でゲーム開始できる
- 8人でゲーム開始できる
- 役職未配布がある人数で正しく配布される
- 名前重複で参加拒否される
- 開始後参加できない
- ホストだけが役職配り直しできる
- BOOM公開で即終了する
- 解除全達成で即終了する
- 4ラウンド終了でボマー団勝利になる
- 同端末リロード復帰できる
